# About Context Files

Context files are saved every day or every few days to capture decisions, progress, and rationale as the project evolves. They are **point-in-time snapshots** — not every detail reflects the current state of the project. Their purpose is to track the evolution of thinking, decisions, and the build.

## How to read them

- Each file covers a date range (e.g. April 14, April 15, April 16-17)
- They capture: what was built, what was decided, what changed, and why
- Later context files may override or update decisions from earlier ones
- For the current state of the project, check [[scope]] and the code repo directly

## How they're created

At the end of each Claude Code session (or when context limits are approached), a context file is generated capturing everything significant from that session. These are also stored in the code repo at `contexts/`.
